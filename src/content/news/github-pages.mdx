## 1. サイトのディレクトリ構成

サイトを構築する際は、以下のようなディレクトリ構成を作成します。
GitHub Pagesで正しく表示させるためには、**リポジトリの直下（ルート）に `index.html` を配置する**必要があります。

保守性や開発体験を高めるためにはReactなどのモダン技術を採用する方法もありますが、今回はWebの基本構造を理解するための最小構成としています。

```text
my-website/
  ├── index.html       (サイトのトップページ)
  ├── about.html       (別ページ)
  ├── css/
  │   └── style.css    (スタイルシート)
  └── images/
      └── profile.jpg  (画像素材)
```

サンプルコードは以下のようになります。

### index.html

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <link rel="stylesheet" href="./css/style.css" />
    <meta charset="UTF-8" />
    <title>ホーム | サンプルサイト</title>
  </head>
  <body>
    <h1>こんにちは！</h1>
    <p>トップページです。</p>
    <hr />
    <a href="./about.html">Aboutページ(自己紹介)へ行く</a>
  </body>
</html>
```

### about.html

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <link rel="stylesheet" href="./css/style.css" />
    <meta charset="UTF-8" />
    <title>About | 自己紹介</title>
  </head>
  <body>
    <h1>自己紹介ページ</h1>
    <p>Web制作を学んでいます。</p>
    <hr />
    <a href="./index.html">ホームに戻る</a>
  </body>
</html>
```

### css/style.css

```css
body {
  font-family: sans-serif;
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  height: 100vh;
  margin: 0;
  background-color: #f0f4f8;
  color: #333;
}

h1 {
  color: #0070f3;
}
```

## 2. リンクと画像の正しい書き方（パスの指定）

複数ページのサイトを作成する際、よくつまづくのが画像の表示や別ページへのルーティング（パスの指定）です。

GitHub Pagesでサブディレクトリとして公開する場合、URLは https://ユーザー名.github.io/リポジトリ名/ という形式になります。  
そのため、リンクや画像のパスは、環境に依存しにくい相対パスで記述するのが最も安全です。
例えば、`index.html` から `about.html` へのリンクは以下のように書きます。

### CSSの読み込み

```html
<link rel="stylesheet" href="./css/style.css" />
```

### 別ページへの遷移

```html
<a href="./about.html">About</a>
```

### 画像の表示

```html
<img src="./images/profile.jpg" alt="プロフィール画像" />
```

## 3.ローカルからGitHubへのプッシュ手順

ローカル環境でコードが完成したら、以下のGitコマンドを使用してGitHubのリポジトリへプッシュします。

```bash
# 1. プロジェクトフォルダでGitを初期化
git init

# 2. 全てのファイルをステージングしてコミット
git add .
git commit -m "Initial commit: Add standard HTML and CSS"

# 3. メインブランチの名前を 'main' に設定
git branch -M main

# 4. リモートリポジトリ（GitHub）のURLを紐付け
# ※ URLはご自身のリポジトリのものに変更
git remote add origin <https://github.com/ユーザー名/リポジトリ名.git>

# 5. GitHubへプッシュ
git push -u origin main
```

今後、サイトを更新したい時は以下のコマンドを用いてプッシュすることで更新できます。

```bash
# 1. 変更したファイルをステージング（追加）する
git add .

# 2. 変更内容にメッセージをつけて記録（コミット）する
git commit -m "任意のメッセージ"

# 3. GitHubに送信（プッシュ）する
git push
```
